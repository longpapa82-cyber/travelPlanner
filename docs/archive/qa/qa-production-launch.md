# TravelPlanner 프로덕션 출시 QA 마스터 플랜

**작성일**: 2026-03-22
**목표**: versionCode 31 프로덕션 출시 전 완벽한 품질 검증
**원칙**: Zero Defect Launch - 발견된 모든 이슈 수정 후 출시

---

## 📊 QA 실행 개요

### 검수 범위
- **Frontend**: React Native (Expo) - 전체 화면 및 기능
- **Backend**: NestJS API - 모든 엔드포인트 및 비즈니스 로직
- **Security**: 보안 취약점 전수 조사
- **Launch Readiness**: Play Store 출시 준비 상태

### 사용 에이전트
1. **auto-qa**: 전체 앱 기능 자동 검수
2. **security-qa**: 보안 취약점 스캔
3. **feature-troubleshooter**: 핵심 플로우 심층 검증
4. **Playwright**: E2E 자동화 테스트 (있는 경우)
5. **Manual**: 출시 체크리스트 수동 검증

### Go/No-Go 기준
- ✅ **Go**: P0 0건, P1 0건, P2 ≤5건, 테스트 통과율 ≥95%
- ❌ **No-Go**: P0 or P1 존재, 테스트 통과율 <95%

## Phase 1: auto-qa 전체 기능 검수

### 목표
Travel Planner 앱의 **모든 기능**을 체계적으로 검수하고, 버그를 자동으로 발견 및 수정

### 검수 대상 기능 (13개 카테고리)

#### A. 인증 및 계정 (Authentication)
1. **회원가입**
   - 이메일 유효성 검증
   - 비밀번호 강도 체크 (8자 이상, 영문+숫자)
   - 비밀번호 확인 일치 검증
   - 중복 이메일 에러 핸들링
   - UI: placeholder 텍스트 표시 (Bug #6 UI fix 검증)

2. **로그인**
   - 이메일/비밀번호 로그인
   - Google OAuth (프로덕션 게시 상태)
   - Kakao OAuth (이메일 수집 설정)
   - 잘못된 자격증명 에러 처리

3. **소셜 로그인**
   - Google 로그인 플로우
   - Kakao 로그인 플로우
   - Apple 로그인 (iOS only, Phase 2)

4. **비밀번호 관리**
   - 비밀번호 찾기 / 재설정
   - 이메일 발송 확인

#### B. 여행 생성 (Trip Creation) ⭐ **최우선 검증**
1. **AI 자동 생성**
   - SSE 스트리밍 프로그레스 표시
   - SSE 30초 타임아웃 동작 (Bug #6-3)
   - 목적지, 날짜, 인원 입력
   - AI 일정 생성 완료
   - **중복 생성 방지** (Bug #6-4): 버튼 연타 시 1개만 생성
   - **AI 카운트 즉시 업데이트** (Bug #6-1): refreshStatus() await 확인
   - 생성 완료 후 TripDetail 화면 이동

2. **수동 생성**
   - 빈 일정으로 여행 생성
   - 수동 일정 추가

3. **에러 시나리오**
   - 네트워크 오류 처리
   - AI API 실패 시 fallback
   - SSE 중단 시 재시도 로직 (Bug #5 fix)

#### C. 여행 목록 및 필터링 (Trip List)
1. **목록 표시**
   - 대기/진행중/완료 상태별 분류
   - **타임존 기반 상태 전환** (Bug #32): 목적지 시간 기준 확인

2. **필터 및 검색**
   - 상태별 필터 (upcoming/ongoing/completed)
   - 목적지 검색
   - 날짜 범위 필터

3. **정렬**
   - 날짜순, 생성일순

#### D. 여행 상세 및 편집 (Trip Detail)
1. **여행 정보**
   - 목적지, 기간, 인원 표시
   - 날씨 정보 표시
   - 타임존 정보 표시

2. **일정 관리**
   - 일정(Itinerary) 추가/수정/삭제
   - 활동(Activity) 추가/수정/삭제
   - 순서 변경

3. **편집 권한**
   - **진행중/완료 여행 편집 제한** (타임존 기반)
   - 과거 일정 편집 제한

4. **공유 및 협업**

## Phase 2: security-qa 보안 점검

### 목표
OWASP Top 10 및 모바일 앱 보안 취약점 전수 조사

### 검수 항목

#### A. 인증 및 세션 (Authentication & Session)
1. **비밀번호 보안**
   - bcrypt 해싱 검증
   - Salt 사용 확인
   - 최소 길이 강제 (8자)

2. **JWT 토큰**
   - 토큰 만료 시간 적절성
   - Refresh token 보안
   - 토큰 저장 위치 (SecureStore)

3. **OAuth 보안**
   - CSRF 방어
   - State 파라미터 사용
   - Redirect URI 검증

#### B. 개인정보 보호 (PII Protection)
1. **데이터 노출**
   - API 응답에 비밀번호 미포함
   - 민감 정보 로그 미출력
   - 에러 메시지에 민감 정보 미포함

2. **데이터 암호화**
   - HTTPS 강제
   - 저장 데이터 암호화

#### C. 접근 제어 (Authorization)
1. **권한 검증**
   - 모든 API 엔드포인트 인증 확인
   - 사용자별 리소스 접근 제한
   - Admin 엔드포인트 권한 검증

2. **IDOR 방어**
   - tripId 소유권 검증
   - userId 필터링

#### D. 입력 검증 (Input Validation)
1. **SQL Injection**
   - TypeORM 파라미터화 쿼리 사용
   - Raw 쿼리 검증

2. **XSS 방어**
   - HTML 이스케이프
   - stripHtml 적용 (Bug #QA-Day3 fix)

3. **파일 업로드**
   - 파일 타입 검증
   - 파일 크기 제한

#### E. Rate Limiting
1. **API Rate Limit**
   - 로그인: 5회/분
   - 회원가입: 3회/시간
   - AI 생성: 사용자 할당량

2. **에러 로그**
   - Rate limit: 100/분 (Bug #Admin fix)

#### F. 제3자 서비스 보안
1. **API 키 보호**
   - 환경 변수 사용
   - 클라이언트 노출 방지

2. **외부 API**
   - OpenAI, Google Maps, LocationIQ 등
   - 에러 처리

### 실행 방법
```bash
# security-qa 에이전트 실행
Task tool with subagent_type: "security-qa"
```

### 성공 기준
- ✅ OWASP Top 10: 모두 PASS
- ✅ Critical/High 취약점: 0건
- ✅ Medium 취약점: ≤3건 (수정 계획 있음)

## Phase 3: feature-troubleshooter 핵심 플로우 검증

### 목표
**End-to-End 사용자 여정**을 실제 시나리오로 심층 검증

### 검증 시나리오

#### 시나리오 A: 신규 사용자 온보딩
```
1. 앱 설치 및 실행
2. 온보딩 슬라이드 확인
3. 이메일 회원가입 (Bug #6 UI fix 검증)
   - placeholder: "비밀번호 (8자 이상)" 전체 표시 확인
4. 이메일 인증
5. 프로필 정보 입력
6. 메인 화면 진입
```

#### 시나리오 B: AI 여행 생성 (전체 플로우) ⭐ **최우선**
```
1. "새 여행" 버튼 클릭
2. 목적지 입력: "뉴욕"
3. 날짜 선택: 2026-04-01 ~ 2026-04-07
4. 인원 선택: 2명
5. "AI 생성" 버튼 클릭

[Bug #6 검증 포인트]
6. SSE 스트리밍 프로그레스 표시
   - 30초 이내 완료 확인
   - 타임아웃 시 적절한 에러 처리
7. 버튼 연타 테스트 (5회 빠르게 클릭)
   - 1개만 생성되는지 확인
8. AI 카운트 즉시 업데이트 확인
   - 3/3 → 2/3 (재시작 없이)
9. 생성 완료 후 TripDetail 화면 자동 이동
10. 일정 확인 (7일치 일정 생성 확인)
```

#### 시나리오 C: 여행 진행 (타임존 검증) ⭐
```
[국내 여행 - 타임존 동일]
1. 서울 여행 생성 (오늘부터 3일)
2. 상태 확인: "진행중" (서울 시간 기준)
3. 첫날 일정 완료 체크
4. 진행률 계산 확인

[국제 여행 - 타임존 차이]
1. 뉴욕 여행 생성 (내일부터 5일)
   - 시차: -14시간 (서울 기준)
2. 상태 확인: "대기" (뉴욕 시간 기준)
3. 시간 경과 시뮬레이션
   - 뉴욕 시간 자정 도달 시 "진행중" 전환 확인
   - 서울 시간 자정이 아닌 뉴욕 시간 기준 검증
4. 여행 종료 시 "완료" 전환 확인
```

#### 시나리오 D: 구독 및 결제
```
1. AI 생성 3회 사용 (0/3 도달)
2. "구독하기" 버튼 노출 확인
3. Premium 플랜 선택
4. RevenueCat 결제 플로우
5. 구독 완료 후 무제한 AI 생성 가능 확인
```

#### 시나리오 E: 에러 복구
```
1. 네트워크 오프라인 상태에서 여행 생성 시도
2. 적절한 에러 메시지 표시
3. 네트워크 복구 후 재시도 성공
```
